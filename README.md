# Midlicense

**A permissive open source license between MIT and GPL - maximum freedom, just can't resell raw code.**

## What is Midlicense?

Midlicense gives you almost total freedom with your code - use it commercially, modify it, sell products built with it - with just one restriction: **you can't resell the raw source code as a standalone product**.

Perfect for:
- Training scripts and ML tools
- Libraries and frameworks  
- Utilities and CLI tools
- Any code where you want maximum adoption but don't want lazy resellers

## Quick Comparison

## FAQ

### Can I use this commercially?
**Yes!** Use it in products, services, at your company - all completely fine.

### Can I sell models/outputs created with code under this license?
**Yes!** Train models, generate outputs, build services - sell them freely. The restriction only applies to reselling the raw source code itself.

### Can I include this in a larger product I'm selling?
**Yes!** As long as the code is part of something bigger (a product, service, framework), not just repackaged and sold alone.

### Do I need to include a copyright notice?
**No**, it's optional. The license works either way. Add one if you want attribution.

### What counts as "substantially similar form"?
Minor formatting changes, renaming variables, or adding trivial comments doesn't make it "substantially different." Real modifications with added functionality or significant restructuring would be fine.

### Is this OSI approved?
**No**, and it won't be - the non-commercial-code clause disqualifies it from OSI approval. But it's still open source in spirit: the code is public, forkable, and usable for nearly everything.

### Why not just use MIT or Apache?
MIT and Apache allow anyone to take your exact code, repackage it, and sell it as their own product with zero modifications. Midlicense prevents that specific case while keeping everything else permissive.

### Why not use GPL?
GPL forces all derivative works to be open source (copyleft). Midlicense lets people build proprietary products using your code - they just can't resell your code itself.

## Why Midlicense?

Existing licenses are either:
- **Too permissive** (MIT/Apache allow code reselling)
- **Too restrictive** (GPL forces derivatives open, CC BY-NC blocks all commercial use)

Midlicense fills the gap: **maximum freedom with minimal protection** against lazy code resellers.
